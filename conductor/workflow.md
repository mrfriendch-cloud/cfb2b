# Conductor Workflow - B2B Product Exhibition Website

This file defines the project's development workflow.

## 1. Development Process
- **Phase-Based Progress:** Features are implemented in logical phases (e.g., Research, Implementation, Verification).
- **Task Management:** Every track is broken down into specific tasks and sub-tasks.

## 2. Testing Strategy
- **Coverage:** Aim for >80% code coverage where applicable.
- **Verification:** Mandatory manual verification by the user at the end of each phase.

## 3. Commit Strategy
- **Frequency:** Commit after each completed task.
- **Summaries:** Record task summaries in the commit message body.

## 4. Phase Completion Protocol
- Each phase must end with a verification task: `- [ ] Task: Conductor - User Manual Verification '<Phase Name>'`.
